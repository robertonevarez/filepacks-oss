filepacks is a narrow OSS tool for turning a directory of files into a deterministic `.fpk` artifact — a single portable file you can inspect, verify, and compare against another run.

It is useful wherever you produce repeated file output and want a stable, reviewable evidence boundary: agent runs, eval outputs, CI build artifacts, regression bundles.

## What a `.fpk` artifact gives you

- One stable file instead of a loose output directory
- A canonical `manifest.json` as the first archive entry with full file inventory
- Payload files stored under `payload/`
- Deterministic archive bytes for the same logical input
- Structural comparison based on manifest entries, not ad hoc directory diffs

## CLI and library

Use the CLI when you want a command-line workflow:

```bash
filepacks pack ./run --output ./run.fpk
filepacks inspect ./run.fpk
filepacks verify ./run.fpk
filepacks compare ./baseline.fpk ./run.fpk
```

Use `@filepacks/core` when you want to build around the artifact format from code:

```ts
import {pack, inspect, verify, compare} from '@filepacks/core'

await pack({input: '/tmp/run', output: '/tmp/run.fpk'})
const artifact = await inspect({artifact: '/tmp/run.fpk'})
const result = await verify({artifact: '/tmp/run.fpk'})
const diff = await compare({
  baseline: '/tmp/baseline.fpk',
  candidate: '/tmp/run.fpk',
})
```

## Public OSS surface

The public CLI and library include:

- `pack` — create a deterministic `.fpk` artifact from a directory
- `inspect` — read artifact metadata from a `.fpk` file
- `verify` — check the payload against the manifest, exit non-zero on mismatches
- `compare` — structural comparison of two artifacts, exit `20` when they differ

The public OSS surface does not include registry workflows, local store management, tags, typed eval manifests, or hosted product behavior.
